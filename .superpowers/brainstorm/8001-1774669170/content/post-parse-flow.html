After parsing — what's the UX?

The user drops a PDF. The API extracts their name, headline, experiences, skills, etc. Then what?

A

Review & Confirm step

Show the parsed data in a preview card before anything is saved. User sees what was extracted, can deselect fields they don't want, then hits "Import to Profile". Clean, trustworthy, no surprises.

Pros

  • User stays in control
  • Handles parse errors gracefully
  • Works even when profile already has data

Cons

  • One extra step
B

Auto-fill empty fields only

Immediately merge parsed data into the profile — only writing fields that are currently blank. Existing data is never overwritten. User lands on the profile editor pre-populated.

Pros

  • Zero friction for new users
  • Fast path to the editor

Cons

  • Silent merge — unclear what changed
  • Risky if parse quality is poor
C

Overwrite & go to editor

Replace the profile entirely with parsed data. Redirect to the profile editor so the user can review and fix from there.

Pros

  • Simplest to implement
  • Editor is the review step

Cons

  • Destroys existing profile data
  • Bad for returning users
